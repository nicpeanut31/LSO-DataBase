WHAT CHANGED
------------
ATTENDANCE
- First Semester and Second Semester are separate ledgers.
- Events now require a Semester field.
- Calendar, overall totals, percentages, individual lookup, event roster, and
  printable reports use only the selected semester.
- Older events without a semester are treated as First Semester and may be
  edited when they belong in Second Semester.

DUTY HOURS
- Duty Hours now has its own navigation page instead of appearing inside
  Attendance.
- First Semester and Second Semester are separate.
- Trainee and Probationary rosters are displayed in separate lists.
- Committed time and incentive adjustments use exact Hour and Minute inputs.
- Rendered duty uses clock-based Time In and Time Out and computes the exact
  duration automatically, including minutes.
- Incentives have their own form:
    Add credit = reduce remaining time
    Deduct credit = increase remaining time
- Calculations are stored as whole minutes to prevent decimal rounding errors.
- Individual, semester roster, and academic-year totals are printable.
- Existing decimal-hour records are preserved and migrated into First Semester.

DASHBOARD
- Separate attendance signals for First and Second Semester.
- Duty Hours signal for both semesters.
- Functional shortcuts to Attendance and Duty Hours.
- Upcoming events display their assigned semester.

USING ATTENDANCE
----------------
1. Open Attendance.
2. Select 1st Semester or 2nd Semester.
3. Create or edit an event and assign the correct semester.
4. All calculations and printouts remain inside the selected semester.

USING DUTY HOURS
----------------
1. Open Duty Hours from the sidebar.
2. Select 1st Semester or 2nd Semester.
3. Choose a name from the Trainee or Probationary roster.
4. Set committed Hour and Minute.
5. In Add Clock-Based Duty, select the date, Time In, and Time Out.
6. The computed duration appears immediately. Click Add Render Time or press Enter.
7. Example: 12:00 PM to 5:00 PM is automatically recorded as 5 hours.
8. Add an incentive using the separate Incentive form.
9. Use Print Individual or Print Roster Report for A4-ready documents.

CLOCK RULE
----------
Time Out must be later than Time In because each duty entry is treated as a
same-day record. Existing manual rendered-time entries remain available and are
shown as Manual duration in the ledger.

SECURITY
--------
The publishable key in supabase-config.js is intended for browser use.
Never put a Supabase secret or service_role key in GitHub.

ATTENDANCE REPORT UPDATE
------------------------
No additional SQL change is required for the Semester and Monthly report buttons.
After uploading all updated files to GitHub, hard-refresh the live website.
Monthly reports use the month currently displayed in the Attendance Calendar.

DUTY HOURS ARCHIVE UPDATE
-------------------------
This package adds Active Records and Duty Archive views. Duty records remain accessible after a member progresses, and skipped Probationary records are marked Skipped • Archived while remaining included in totals. No new Supabase SQL is required when the current Semester Duty Hours setup is already installed.
